# تعليمات نشر الموقع على GitHub Pages

## المشكلة التي تم حلها
كان موقعك يظهر شاشة بيضاء لأنه كان يحتوي على الكود المصدري فقط. تحتاج GitHub Pages إلى ملفات ثابتة (static files) مُبنية مسبقاً.

## الحلول المطبقة

### 1. تحديث إعدادات Vite
✅ تم تحديث `vite.config.ts` لدعم GitHub Pages
✅ إضافة `base: '/markt/'` للمسارات الصحيحة
✅ تحسين عملية البناء

### 2. بناء المشروع بنجاح
✅ تم إنشاء مجلد `dist` مع جميع الملفات المطلوبة
✅ تم ضغط وتحسين الملفات للنشر

### 3. إعداد النشر التلقائي
✅ تم إنشاء GitHub Actions workflow للنشر التلقائي

## خطوات النشر على GitHub

### الخطوة 1: رفع الملفات المحدثة
```bash
# تأكد من رفع هذه الملفات إلى GitHub:
- .github/workflows/deploy.yml
- vite.config.ts (المحدث)
- dist/ (مجلد البناء)
```

### الخطوة 2: إعداد GitHub Pages
1. اذهب إلى إعدادات repository على GitHub
2. اختر "Pages" من القائمة الجانبية
3. في "Source" اختر "GitHub Actions"
4. احفظ الإعدادات

### الخطوة 3: تفعيل GitHub Actions
1. اذهب إلى تبويب "Actions" في repository
2. فعّل workflows إذا كانت معطلة
3. اقترح workflow جديد إذا لم يظهر تلقائياً

### الخطوة 4: اختبار النشر
1. اضغط على الكود إلى branch `main`
2. انتظر حتى ينتهي workflow (حوالي 2-3 دقائق)
3. اذهب إلى رابط الموقع: `https://username.github.io/markt/`

## ملاحظات مهمة

### قاعدة URL
- تأكد من أن اسم repository هو `markt` أو غيّر `base` في `vite.config.ts`
- إذا كان اسم repository مختلف، غيّر السطر في `vite.config.ts`:
  ```typescript
  base: '/اسم-المستودع/', // غير هذا الاسم
  ```

### فحص البناء المحلي
لاختبار البناء محلياً قبل الرفع:
```bash
npm install
npm run build
# ثم افتح dist/index.html في المتصفح
```

## استكشاف الأخطاء

### إذا لم يظهر الموقع:
1. تحقق من أن GitHub Actions تم تشغيله بنجاح
2. تأكد من أن اسم repository يطابق `base` في config
3. تحقق من أن GitHub Pages مُفعل في الإعدادات

### إذا ظهرت أخطاء JavaScript:
1. افتح Developer Console في المتصفح
2. تحقق من أخطاء تحميل الملفات
3. تأكد من أن المسارات صحيحة
